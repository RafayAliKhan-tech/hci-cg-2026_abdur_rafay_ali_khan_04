Purpose of This Page

This page provides a clear comparison of Norman’s Action Cycle, GOMS, and the Keystroke-Level Model (KLM). The goal is to show how these models differ in what they explain and predict, as required in Lab & Assignment 01.

Model Comparison Overview

Norman, GOMS, and KLM: explanation vs prediction
Model Type What It Explains What It Predicts
Norman Descriptive User goals, actions, feedback, and errors ❌ No time or performance prediction
GOMS Analytic / predictive Task structure for expert users ✔ Relative efficiency of methods
KLM Quantitative Low-level execution actions ✔ Execution time

KLM Operators

The Keystroke-Level Model decomposes interaction into primitive operators. It assumes expert, error-free performance and focuses only on execution.

KLM operators (Dix et al., Chapter 12)
Symbol Name Description Example
K Keystroke Pressing a key Typing a letter
B Button Mouse button press Left-click
P Pointing Moving pointer to a target Move cursor to icon
H Homing Switching hand between devices Keyboard → mouse
M Mental Mental preparation Deciding to click
R Response System response time App loading

Worked Example (KLM)

Task: Correct a single typing error in a text editor.

  1. Move hand to mouse
  2. Position cursor
  3. Return to keyboard
  4. Delete incorrect character
  5. Type correct character

Possible KLM sequence:

H  P B
H  M K
K

This example shows that KLM models execution only. Planning and interpretation are outside its scope.

Key Takeaways

Norman explains why users succeed or fail, GOMS structures tasks for expert users, and KLM predicts how long execution will take.
