Don Norman and Human–Computer Interaction

Don Norman is one of the most influential researchers in the field of Human–Computer Interaction (HCI). His work focuses on how humans interact with everyday objects and digital systems.

In The Design of Everyday Things, Norman explains that good design should make actions visible, reduce errors, and match the user’s mental model.

  • Affordances: Properties that suggest how an object can be used.
  • Signifiers: Visual indicators that show where actions should occur.
  • Feedback: Information returned to the user after an action.
  • Constraints: Limitations that prevent incorrect usage.

Applying Don Norman’s Principles Using HTML

HTML can be used to support Don Norman’s usability principles by providing structure, clarity, and accessibility to web content.

Norman’s Principle HTML Implementation
Visibility Using headings (<h1>–<h6>) to show content hierarchy
Feedback Confirmation messages after form submission
Affordances Buttons (<button>) clearly indicate clickable actions
Constraints Required attributes in input fields

Common HTML Errors and Usability Impact

Poor HTML practices can negatively affect usability and violate Don Norman’s design principles.

  1. Missing alt attribute: Breaks accessibility and feedback for visually impaired users.
  2. Improper heading order: Confuses users and damages mental models.
  3. Non-semantic HTML: Reduces clarity and discoverability of content.
  4. No meta viewport: Causes poor usability on mobile devices.
  5. Unlabeled form fields: Violates visibility and feedback principles.

Conclusion

Don Norman’s principles provide a strong theoretical foundation for designing usable and user-centered interfaces. When applied correctly, HTML supports these principles by offering structure, accessibility, and clarity.
