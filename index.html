Biography

Don Norman is a leading cognitive scientist and pioneer of Human–Computer Interaction (HCI) and User Experience (UX) design. He has worked at Apple, Northwestern University, and the University of California, San Diego. He is best known for "The Design of Everyday Things", which explains that user errors are usually caused by poor design, not by users.

Core Design Principles
  • Affordances: Physical properties suggesting how to use an object.
  • Signifiers: Visual cues showing where interaction should occur.
  • Feedback: Clear communication about the result of an action.
  • Constraints: Limits that prevent incorrect actions.
Norman's Seven Stages of Action
  1. Forming the goal
  2. Forming the intention
  3. Specifying the action
  4. Executing the action
  5. Perceiving the system state
  6. Interpreting the system state
  7. Evaluating the outcome

Example: Using a Toggle Switch – user forms the goal, decides to switch it on, executes the action, sees the state change, interprets feedback, and evaluates success.

Gulfs of Interaction

Gulf of Execution: The gap between what the user wants to do and what the system allows.

Gulf of Evaluation: The gap between what the system does and how well the user understands it.

Good design minimizes both gulfs using clear actions, visible feedback, and simple interfaces.

HCI Insights

Norman emphasizes that blaming users for errors is wrong; design should support error-free interactions.

Simple, clear interfaces help users form correct mental models and reduce mistakes.

Example: In an Automatic Syringe, clear labels and indicators guide the user step by step, reducing the gulf of execution.

The Father of UX

"Good design is actually a lot harder to notice than poor design, in part because good designs fit our needs so well that the design is invisible."

— DON NORMAN
