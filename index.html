Don Norman and Human–Computer Interaction

Don Norman is a prominent figure in the field of Human–Computer Interaction (HCI). His work focuses on understanding how users interact with systems and how design can reduce human error.

According to Dix et al., HCI is concerned with the design, evaluation, and implementation of interactive computing systems for human use. Norman’s ideas strongly support this definition by emphasizing usability and user-centered design.

  • Affordances: Properties that indicate possible actions.
  • Signifiers: Cues that communicate where actions should occur.
  • Feedback: Information returned after an action.
  • Constraints: Restrictions that prevent incorrect usage.

HCI Concepts Explained Through HTML

HTML provides the structural foundation of web interfaces. When designed correctly, HTML supports HCI principles such as visibility, consistency, and feedback.

HCI Principle HTML Example
Visibility Clear headings using <h1>–<h6>
Feedback Confirmation messages after user actions
Consistency Repeated navigation structure on all pages
Constraints Using required fields in forms

Common HTML Errors and Usability Problems

Interaction Design emphasizes that poor implementation leads to poor user experience. The following HTML errors negatively affect usability.

  1. Missing alt text in images: Breaks accessibility and violates usability principles.
  2. Improper heading structure: Confuses users and disrupts mental models.
  3. Non-semantic HTML: Reduces clarity and understandability.
  4. No viewport meta tag: Causes poor interaction on mobile devices.
  5. Unclear navigation: Violates consistency and discoverability.

Conclusion

The principles discussed by Don Norman align closely with the foundations of Human–Computer Interaction described by Dix et al. and Interaction Design literature. Proper use of HTML supports these principles by improving structure, clarity, and usability.
