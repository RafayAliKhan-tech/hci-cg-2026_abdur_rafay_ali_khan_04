Biography

Don Norman is a leading cognitive scientist and pioneer of Human–Computer Interaction (HCI) and User Experience (UX) design. He is best known for his book "The Design of Everyday Things", which explains that user errors are usually caused by poor design, not by users.

Core Design Principles
  • Affordances: Physical properties suggesting how to use an object.
  • Signifiers: Visual cues showing where interaction should occur.
  • Feedback: Clear communication about the result of an action.
Norman's Seven Stages of Action
  1. Forming the goal
  2. Forming the intention
  3. Specifying the action
  4. Executing the action
  5. Perceiving the system state
  6. Interpreting the system state
  7. Evaluating the outcome
Gulfs of Interaction

Gulf of Execution: The gap between what the user wants to do and what the system allows.

Gulf of Evaluation: The gap between what the system does and how well the user understands it.

Good design minimizes both gulfs with clear actions and meaningful feedback.

HCI Insights

Don Norman emphasizes that users should not be blamed for mistakes; poor design is usually the cause.

These principles help users build correct mental models and improve usability.

The Father of UX

"Good design is actually a lot harder to notice than poor design, in part because good designs fit our needs so well that the design is invisible."

— DON NORMAN
